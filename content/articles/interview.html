Principles

  • Try to ask questions in a form of a challenge.

    "Here is a business need ... how you would handle this need?"

  • If there is a several people to interveiew - select question blocks random and different way. But stil you need to cover all type of questions and be able to compare them

Begining

Introduce myself

If you don't mind, let's start our conversation in English. That way we could clarify you English level at the begining.

"About me"

  • Could you tell me about there project you contrebuted the most? What were your resposibilities in the project? What about the most interesting prject you have been working on? What role did you have on it?
  • In what project roles do you have an experience? (DevLead, TechLead, QA, Manager)

Before start

Because we have limited time and a lot of topics to discuss, it could happend that I will interrupt and stop you. I hope you understand.

Web

Have you ever started Web project from scratch? (Commercial)

What parts of Web application you can differentiate? (We can talk about imaginary application, and go from simple to complex)

  • Client (HTML templates, jQuery, SPA / Angular / React )
  • Web server (Nginx / Apache / LiteSpeed Web Server / Lighttpd / Caddy)
  • WSGI server (uWSGI / GUnicorn)
  • Backend Framework (Django, FastAPI, ExpressJs)
  • Communication protocol. REST / GraphQL / XML / RPC / ... What do you like more, for what kind of purpose what protocols are appropriate? What Pros/Cons do you see?
  • Cache (Redis / Memcache)
  • Database (MySQL, Postgres, MongoDB) (ORM ?, SQLAlchemy, DjangoORM)
  • Logs (What to log, where to log? How you would work with them later?) Metrics, Alerts (ELK/EFK)
  • Search in a lot of data (Advertisement site, Internet shop). Elasticsearch
  • Tests
  • Distributed Task Queue (Celery)
  • MQ / AMQP (RabbitMQ, Kafka, ZeroMQ, Redis, NATS)
  • API docs / Swagger / OpenAPI
  • Docs / Confluence
  • ETL (Extract-Transform-Load)

With which of them you have worked with? Which of them are the most interesting?

HTTP

  • What happends when we put some URL in browser and press 'Enter'.
  • What can you say about OSI model | TCP/IP model

  • What parts HTTP reqest has?

    • Actions / Headers / Body
    • Difference between GET and POST

  • REST (Representational state transfer) / RESTful API (All constraints are satisfied).
    Uniform interface (Identification of resources, Manipulation of resources through representations, Self-descriptive messages, Hypermedia as the engine of application state), Client-server, Stateless, Cacheable, Layered system, Code on demand (optional)
    Resource concept (Uniform Resource Identifiers / URIs) and resource representation: data, metadata, hypermedia links. Naming
    Resource Methods and your typical implementation of them

    Best Practices

    Use nouns to represent resources (resource archetypes : document, collection, store, and controller) /
    • Best Practices

    https://restfulapi.net/
  • SOAP
  • Have you used GraphQL? Have you implement GraphQL?
  • CORS Headers (Cross-Origin Resource Sharing)
  • TLS (Transport Layer Security), SSL (Secure Socket Layer)

    Git

    • Why do we need it? How do you use Git? Tell me about agreements you have used. (Git flow, branches)
    • How good PR looks like?
    • What instruments you use to manage git repo?
    • What are merge conflicts?
    • Have you ever done Force push?
    • Have you used 'rebase'(interactive) in git?
    • Have you had an experience with SVN / Mercurial?

    Databases

    • What types of databases do you know? (SQL/NoSQL/Key-Value)
    • How good do you know SQL? SQL questions with HAVING, indexes, types of indexes?
    • What is ORM? Vs raw SQL?
    • When do we need to use JOIN/Indexes? There is a table with authors and the one with books. How would you create a SQL query to show a list of such elements as next: Author name and his life dates with all books he has written.
    • What is 'primary key' / 'foreign key'?
    • Why do we need transactions?
    • Have you ever migrated your DB schema/table?
    • With what DBs have you worked? (PostgreSQL, MySQL, MariaDB, MongoDB, Redis, Memcache)
    • How to check DB performance? ('Explain' functionality?)

    Soft skills / Personality Traits

    • English level (If not discussed at the begining)
