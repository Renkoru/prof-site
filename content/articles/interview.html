Принципы

  • Пытаться ставить вопросы в виде постановки проблемы.

    "Есть такая задача, что бы вы использовали, чтобы с ней справиться?"

  • Если несолько интервьюированных, то выбирать блоки рандомно и разные, но чтобы людей в итоге оценить полно и можно было их сравнить.

Before start

Так как время у нас ограниченно, а вопросов которые хочется обсудить довольно много - иногда мне придётся вас перебивать и останавливать, я надеюсь на ваше понимание.

Web

Начинали ли вы когда-нибудь Web проект с нуля? (Комерческий)

Из каких частей состоит Web приложение?

  • Client (HTML templates, jQuery, SPA / Angular / React )
  • Web server (Nginx / Apache / LiteSpeed Web Server / Lighttpd / Caddy)
  • WSGI server (uWSGI / GUnicorn)
  • Backend Framework (Django, Flask, Asyncio/aiohttp, FastAPI)
  • Communication protocol. REST / GraphQL / XML / RPC / ... Что вам ближе, наравится больше, для каких задач больше подходит?
  • Cache (Redis / Memcache)
  • Database (MySQL, Postgres, MongoDB) (ORM ?, SQLAlchemy, DjangoORM)
  • Logs (Куда писать, Что нужно логировать? Как потом это использовать) Metrics, Alerts (ELK)
  • Search in a lot of data (Объявления, Интернет магазин). Elasticsearch
  • Tests
  • Distributed Task Queue (Celery)

Над какими из них вам приходилось работать. Что интереснее всего?

HTTP

  • What can you say about OSI model | TCP/IP model

  • What parts HTTP reqest has?

    • Actions / Headers / Body
    • Difference between GET and POST

  • REST (Representational state transfer) / RESTful API (All constraints are satisfied).
    Uniform interface (Identification of resources, Manipulation of resources through representations, Self-descriptive messages, Hypermedia as the engine of application state), Client-server, Stateless, Cacheable, Layered system, Code on demand (optional)
    Resource concept (Uniform Resource Identifiers / URIs) and resource representation: data, metadata, hypermedia links. Naming
    Resource Methods and your typical implementation of them

    Best Practices

    Use nouns to represent resources (resource archetypes : document, collection, store, and controller) /
    • Best Practices

    https://restfulapi.net/
  • Have you used GraphQL? Have you implement GraphQL?
  • CORS Headers (Cross-Origin Resource Sharing)
  • TLS (Transport Layer Security), SSL (Secure Socket Layer)
  • Web sockets.

Databases
